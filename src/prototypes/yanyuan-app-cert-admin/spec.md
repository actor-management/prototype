# 认证审核管理台 (yanyuan-app-cert-admin)

## 📋 业务与功能

### 1.1 核心目标
>
> 面向协会管理员（Admin）的移动端一站式认证审核工作台SPA。此原型将融合“认证审核台列表”与“审核详情”于单一组件内，无缝衔接列表筛选与深入查看。用于管理员在移动端快速处理演员（包含所有职业详细资料）与普通用户（仅实名结果）的不同认证申请，提升流转与审批效率。

### 1.2 功能清单

- **模块 1：审核台首页与列表**
  - **业务统计面板**：横向数据卡片（本月待审核、已通过、已驳回）。
  - **状态分类导航**：全部 / 待审核 / 已通过 / 已驳回的快捷切换。
  - **动态审核列表**：根据申请人身份（演艺人员/普通用户）和当前状态展示卡片（包含核心身份标识与申请时间）。
- **模块 2：认证申请详情页（内嵌SPA跳转）**
  - **动态详情内容**：若为 Actor，显示丰富的“艺名”、“从业方向”、“教育背景”、“附件材料”等折叠表单；若为 User，仅显示基本身份、实名及认证活体结果。
  - **审核历史跟踪**：显示每次的系统或人工审批动作时间线。
- **模块 3：审核操作面板**
  - **通过与驳回动作**：置底悬浮按键。点击驳回弹出需填原因的毛玻璃 Bottom Sheet。

### 1.3 交互要点

- **SPA路由联动**：列表项点击后，整个视图平滑切换（可带动画掩盖）至详情状态；详情页左上角点击返回恢复列表视图。
- **图片全屏预览**：对于用户的证件及演艺作品附件，点击缩略图则黑色全屏放大预览。
- **驳回输入验证**：必须填写超过一定字数（或有文本）才可点亮确认驳回。

## 📊 内容规划

### 2.1 信息架构

```
认证审核管理台 (yanyuan-app-cert-admin)
├── 视图A：列表页 (ListView)
│   ├── 顶部统计数据组件 (Stats Cards)
│   ├── 滚动吸顶的分类导航 (Tabs)
│   └── 用户申请卡片群列表 (Card List)
│       └── 单条申请卡片 (姓名 / 角色类型 / 时间 / 快速进入角标)
├── 视图B：详情页 (DetailView)
│   ├── 顶部操作导航 (含返回上级)
│   ├── 用户核验摘要 (实名活体检测 100%)
│   ├── 动态折叠卡组 (Accordions)
│   │   ├── Actor 专有项：艺名/从业/联系/教育/特级资质上传图
│   │   └── 全体通用项：实名身份证验证结果
│   └── 审批操作悬浮底栏 (Passed / Rejected Actions)
└── 全局辅助组件
    ├── 图片原图放大层 (Image Lightbox)
    └── 驳回理由输入底层弹窗 (Bottom Sheet)
```

### 2.2 数据来源

- **数据结构**：自包含复杂的树形JSON Mock。
- **关键字段**：
  - `applicant_type`: `actor` | `user` (标识显示不同层面的表单)
  - `status`: `pending` | `approved` | `rejected`
  - `baseInfo.stage_name`: 艺名项
  - `materials`: 证件/附件照片 URL 列表

### 2.3 内容示例

```json
{
  "id": "A1039",

  "status": "pending",
  "timeline": ["2026-04-01: 提交实名", "2026-04-03: 提交艺人和从业背景"]
}
```

## 🎨 布局与结构

### 3.1 整体布局

- **布局模式**：全屏自适应视图控制器 (`100vw`, `100vh`)，外层容器包裹，内部借助 React State 作为当前路由控制器拆分两个大屏。
- **关键尺寸**：
  - 核心内容区使用 `flex-1` 结合 `overflow-y-auto` 滚动。
  - 详情页表单间距采用统一的 `16px` padding，模块白底卡片带有 `border-radius: 12px` 与极轻量阴影 `shadow-sm`。

### 3.2 响应式适配

- 第一优先适配竖屏移动端尺寸（iPhone 设备标准宽 375-430px）。

## 🎨 视觉规范

### 4.1 设计规范来源

**设计规范来源**：内置 frontend-design 规范（混合管理端的数据整洁度与客户端卡片阴影质感），同时保持平台原本的朱红色/丹砂红调性。

### 4.2 自定义设计要点

- **身份标识标签**：
  - “演艺人员认证” 使用统一的常规分类标签颜色（如淡蓝或灰色），背景 `bg-blue-50` 文本 `text-blue-600` 等经典搭配，禁止使用不协调的深紫色。
  - “普通用户认证” 使用低饱和度的灰色（体系统一）。
- **审批动作视觉引导**：底部通过使用品牌统一规范色彩布局体系。
- **底层边界限制**：移动端底部固定菜单及弹窗在PC预览时必须遵循 `max-w-[430px] mx-auto` 的宽度居中边界约束。

### 4.3 组件状态

- **Bottom Sheet (驳回面板)**：上拉抽出动画，背景蒙尘遮罩 `rgba(0,0,0, 0.4)`，点击蒙版空白可快速关合。

## ⚙️ Axure API 说明

### 5.1 事件列表（eventList）

| 事件名称 | Payload 类型 | 触发时机 | 说明 |
|---------|-------------|---------|------|
| `on_item_click` | `string` | 审核卡片"查看详情"被点击 | 返回申请 ID |
| `on_action_reject` | `string` | 管理员成功填写并提交了驳回动作 | 返回驳回的用户ID及原因体 |
| `on_action_pass` | `string` | 管理员确认通过了审核 | 返回所通过的ID |

### 5.2 动作列表（actionList）

| 动作名称 | Params 类型 | 参数说明 | 功能描述 |
|---------|------------|---------|---------|
| `refresh_data` | `void` | - | 重置并返回列表页态 |

### 5.3 变量 / 配置 / 数据

该组件内部已维护数据驱动的复杂列表，对外以 Event 透出结果状态即可。
