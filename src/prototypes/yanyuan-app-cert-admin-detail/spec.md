# 认证审核详情 (yanyuan-app-cert-admin-detail)

## 📋 业务与功能

### 1.1 核心目标
> 演艺人才平台（移动端）管理后端的单据处理详情页。用于管理员在移动场景下深度审查单条演员资质认证申请，查看各类附件，并作出最终判定。

### 1.2 功能清单
- **摘要总览**：申请人核心信息与状态标签
- **身份材料浏览**：大尺寸身份证展示槽，点击可全屏看大图
- **信息折叠查看**：基本、联系、从业、教育信息的分类手风琴面板
- **审核历史跟踪**：时间线组件展示历次提交与驳回记录
- **底部审核操作栏**：固定在屏幕底部的"驳回"与"通过"按钮

### 1.3 交互要点
- 图片点击预览：点击身份证区域，弹出黑色全屏预览层屏蔽背后内容。
- 折叠面板：点击Header可收放卡片内容并变更右侧箭头朝向。
- 驳回弹窗：点击"驳回"，弹出带遮罩的Bottom Sheet输入退回原因。


- **关键字段**：
  - `data.baseInfo`: 基本信息键值对列表
  - `data.timeline`: 包含 `title`, `time`, `desc` 的数组

时间线示例文案：“用户首次提交认证材料” / “原因：身份证反面照片模糊，无法看清有效期”

## 🎨 布局与结构

### 3.1 整体布局
- **布局模式**：全屏 Flex，主体内容 Y 轴滚动。
- **容器设置**：流式 100vw，底栏使用 `fixed` 配合 `padding-bottom` 防遮挡。
- **关键尺寸**：
  - 横向 Padding：16px
  - 详情卡片：圆角 12px，Card内边距 16px
  - 底部操作栏：高度 68px + 安全区

### 3.2 响应式适配
- 同样专为**移动端** App（或微信小程序环境）设计。
- 内容自然堆叠不设多列，大屏设备居中限宽或沿用流式均可，但首要按 iPhone (390px 宽) 视口校准。

## 🎨 视觉规范

### 4.1 设计规范来源
**设计规范来源**：内置 frontend-design

### 4.2 自定义设计要点
**重要元素说明**：
- **全屏黑屏预览**：使用黑色 (`rgba(0,0,0,0.95)`) 覆盖 Z-index 999 渲染，仅显高亮证件与关闭图标。
- **Timeline（时间线）**：首项为红色 `CheckCircle`，历史项目为灰色 Dot。

### 4.3 组件状态
- **Bottom Bar 按钮（通过）**：红色实心背景，高优凸显。
- **Bottom Bar 按钮（驳回）**：浅红背景+红字，警示弱视效。
- **折叠拉开态**：平滑展示内部 Key-Value 内容行。

## ⚙️ Axure API 说明

### 5.1 事件列表（eventList）
| 事件名称 | Payload 类型 | 触发时机 | 说明 |
|---------|-------------|---------|------|
| `on_back` | `void` | 顶部返回图标被点击 | 退回审核列表 |
| `on_pass` | `void` | 底部"通过"按钮被点击 | 更新数据并触发返回 |
| `on_reject` | `void` | "确定驳回"按钮被点击 | 拦截逻辑并确认驳回 |

### 5.2 动作列表（actionList）
| 动作名称 | Params 类型 | 参数说明 | 功能描述 |
|---------|------------|---------|---------|
| `refresh_data` | `void` | - | 重新加载详情项 |

### 5.3 变量 / 配置项
该组件纯数据映射驱动，无独立配置项。
