# 证书申请管理台（PC端）

## 📋 业务与功能

### 1.1 核心目标
> Admin 后台页面，用于管理证书申请全流程。支持按状态 Tab 快速切换查看、多条件筛选、查看申请详情弹窗、审核操作及签发操作。

### 1.2 功能清单
- **Tab 状态切换**：待审核 / 审核中 / 已通过 / 已驳回 / 全部
- **统计卡片**：待审核数、本月申请数、通过率、平均处理时长
- **多条件筛选**：证书类型、证书等级、申请人姓名、申请编号、申请时间范围、审核状态
- **数据表格**：序号、申请编号、申请人姓名、证书类型、证书等级、认定方式、申请时间、当前阶段、审核状态、操作
- **行级操作**：查看（详情弹窗）、审核（pending 状态）、签发（待签发阶段）、催办
- **查看详情弹窗**：申请概要 + 申请人信息 + 认定材料区 + 培训/考核进度 + 审核记录时间线

### 1.3 交互要点
- 「查看」按钮 → 打开申请详情弹窗
- 「审核」按钮 → 打开审核操作弹窗（通过/驳回/需补充材料）
- 「签发」按钮 → 二次确认弹窗后签发证书
- Tab 切换 → 快速过滤不同状态
- 导航菜单 → 跳转已实现页面

## 📊 内容规划

### 2.1 信息架构
```
证书申请管理台
├── 顶部导航栏（与全平台统一）
├── 面包屑（首页 > 证书管理 > 证书申请管理台）
├── 页面标题 + 说明
├── 统计卡片区（4 列）
├── Tab 状态栏
├── 筛选区
├── 数据表格 + 分页
└── 页脚
```

### 2.2 数据来源
- **数据类型**：证书申请列表
- **数据源**：MOCK 示例数据
- **关键字段**：
  - `apply_no`: 申请编号
  - `applicant_name`: 申请人姓名
  - `cert_type`: 证书类型（actor_cert / tech_cert）
  - `cert_level`: 证书等级（level_1 ~ level_4）
  - `ability_proof_type`: 认定方式
  - `current_stage`: 当前阶段
  - `status`: 审核状态

## 🎨 布局与结构

### 3.1 整体布局
- **布局模式**：单栏全宽
- **容器宽度**：最大 1440px 居中
- **关键尺寸**：统计卡片 4 等分，筛选 3 列网格

### 3.2 响应式适配
- **桌面端（≥1200px）**：标准 4 列统计 + 全宽表格

### 4.1 设计规范来源
**设计规范来源**：主题 /src/themes/antd-new（朱红 #B8292F + 暖金 #C5963A 品牌色）

- Tab 激活态使用品牌朱红底色

- **默认态**：标准展示

- **选中态**：Tab 激活高亮
- **弹窗态**：遮罩 + 弹窗动画

## ⚙️ Axure API 说明

### 5.1 事件列表
| 事件名称 | 触发时机 | 说明 |
|---------|---------|------|
| `onReview` | 审核操作提交时 | 携带申请ID和审核结论 |
| `onIssue` | 签发确认时 | 携带申请ID |
| `onNavClick` | 导航菜单点击 | 携带菜单key |

### 5.2 配置项
| 配置项 | 类型 | 默认值 | 说明 |
|--------|-----|-------|------|
| `userName` | string | '管理员' | 当前登录用户名 |
